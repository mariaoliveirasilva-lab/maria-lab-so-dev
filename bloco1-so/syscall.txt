# PARTE C — Questões de síntese

# 1. A apresentação mostrou que "instruções de I/O são privilegiadas". 
#    Com base no que o strace revelou, explique com suas palavras como um programa
#    de usuário consegue fazer I/O sem executar instruções privilegiadas diretamente.

Resposta 1:
Um programa de usuário não possui permissão de hardware para acessar dispositivos de I/O diretamente, pois roda no modo de operação não-privilegiado da CPU (Modo Usuário). Para realizar o I/O, o programa precisa solicitar formalmente esse serviço ao sistema operacional através de uma chamada de sistema (system call), como a write(). Essa chamada gera uma interrupção de software que altera o nível de privilégio da CPU para o Modo Kernel, permitindo que o sistema operacional execute as instruções de I/O com segurança em nome do programa e, em seguida, devolva o controle ao Modo Usuário.

# 2. Compare write(fd, buffer, tamanho) com o "programa de canal"
#    da apresentação: o que os dois têm em comum em termos de
#    responsabilidades?

Resposta 2:Ambos compartilham a responsabilidade de isolar o programador dos detalhes físicos e de baixo nível do hardware, servindo como uma interface de abstração para o transporte de dados. Além disso, tanto a assinatura da função write() quanto um programa de canal precisam definir obrigatoriamente os mesmos três parâmetros fundamentais para uma operação de transferência: o destino/origem do fluxo (representado pelo fd ou pelo identificador do dispositivo), a localização dos dados na memória principal (o buffer ou o endereço de início do canal) e a quantidade exata de informação a ser movimentada (o tamanho em bytes ou o contador de bloco).
